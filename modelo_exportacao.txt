BIGNORDESTE ANALYTICS - MODELO DE EXPORTACAO TXT

Objetivo
Padronizar a exportacao dos resultados das consultas SQL do VR para importacao no sistema.

O sistema trabalha com analise por periodo, mas os arquivos devem vir mensalizados pela coluna mes.
Exemplo de periodo: 2026-01-01 ate 2026-07-01.
Resultado esperado: linhas separadas para 2026-01, 2026-02, 2026-03, 2026-04, 2026-05 e 2026-06.

Formato obrigatorio
- Extensao: .txt
- Separador: ponto e virgula (;)
- Primeira linha: cabecalho com os nomes das colunas
- Encoding recomendado: UTF-8
- Encoding aceito: ISO-8859-1
- Casas decimais: ponto ou virgula sao aceitos pelo sistema
- Campos vazios: aceitos
- Coluna mes obrigatoria em todos os arquivos, no formato YYYY-MM

Ordem recomendada por loja e periodo
1. resumo_geral_loja.sql -> resumo_geral_loja.txt
2. campanhas_ofertas.sql -> campanhas_ofertas.txt
3. cupons_totais.sql -> cupons_campanha.txt
4. departamentos_campanha.sql -> departamentos_campanha.txt
5. produtos_campanha.sql -> produtos_campanha.txt

Consultas oficiais de apoio diario
6. ofertas_dia_campanha.sql -> venda_dia_campanha.txt
7. venda_diaria_loja.sql -> venda_diario.txt

Observacao: os arquivos 6 e 7 servem para validar campanhas pelo periodo real de execucao. Eles podem ser importados como apoio opcional, mas nao sao obrigatorios no checklist mensal dos 5 arquivos.

Checklist de importacao
1. Selecionar loja
2. Selecionar data inicial
3. Selecionar data final
4. Rodar as 5 consultas no VR
5. Exportar .txt separado por ;
6. Importar os 5 arquivos no sistema
7. Validar auditoria
8. Validar consolidacao mensal e acumulada

Parametros que devem ser trocados nas consultas
- :id_loja -> codigo da loja no VR
- :data_inicio -> data inicial do periodo, exemplo 2026-01-01
- :data_fim -> data final exclusiva do periodo, exemplo 2026-07-01

Regra importante sobre datas
Use v.data >= :data_inicio e v.data < :data_fim.
Para analisar janeiro a junho de 2026, use:
- data_inicio = 2026-01-01
- data_fim = 2026-07-01

Expressao obrigatoria para mes
TO_CHAR(DATE_TRUNC('month', v.data), 'YYYY-MM') AS mes

Cabecalhos esperados

resumo_geral_loja.txt
loja;mes;venda_total;custo_total;lucro_total;margem_total;cupons_totais;unidades_totais;produtos_distintos

campanhas_ofertas.txt
loja;mes;id_tipooferta;descricao;venda;custo;lucro;margem;quantidade;itens;produtos_distintos;cupons;ticket_medio;preco_medio

produtos_campanha.txt
loja;mes;id_tipooferta;campanha;id_produto;descricao_produto;quantidade;venda;lucro;margem

departamentos_campanha.txt
loja;mes;id_tipooferta;campanha;mercadologico1;descricao_departamento;produtos;quantidade;venda;lucro;margem

cupons_campanha.txt
loja;mes;cupons_totais;campanha;cupons_campanha
